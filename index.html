What is a Process?

Definition

A Process is a program that is currently executing. It includes:

  • Program code (instructions)
  • Current activity (Program Counter)
  • CPU registers
  • Memory (stack + heap)
  • Resources (files, I/O devices)

Real-life Analogy

Program is like a recipe book.
Process is like cooking the food.

  • Recipe book = stored instructions
  • Cooking = active execution
  • Gas stove + ingredients = resources

Example

When you open Chrome:

  • Chrome.exe = Program
  • Chrome running with tabs = Process
  • Each tab can create separate process

Process Control Block (PCB)

What is PCB?

PCB (Process Control Block) is a data structure maintained by OS. It stores complete information of a process.

OS cannot remember everything about each process, so it stores details in PCB.

PCB Diagram (Main Fields)

Process ID (PID)

Unique number given to process.

Process State

New / Ready / Running / Waiting / Terminated.

Program Counter

Address of next instruction to execute.

CPU Registers

Stores temporary CPU values.

Memory Info

Stack, heap, page table, base limit.

Scheduling Info

Priority, queue pointers, CPU time.

I/O Status

List of open files, devices used.

Accounting Info

CPU usage time, user ID, execution stats.

PCB Real-life Analogy

PCB is like a student record file in college.

  • PID = Student USN
  • Registers = Current marks
  • State = Present/Absent
  • Scheduling info = Class timetable

Process Scheduling (Introduction)

Why Scheduling is Needed?

  • CPU can execute only one process at a time
  • Many processes are waiting in ready queue
  • OS must decide which process gets CPU

Scheduler Types

  • Long-Term Scheduler: selects process from disk to RAM
  • Short-Term Scheduler (CPU Scheduler): selects process from ready queue to CPU
  • Medium-Term Scheduler: swaps process in/out (suspension)

Key Term: Ready Queue

Ready queue is a list of processes waiting for CPU. Scheduler picks one process from ready queue.

Context Switching

Definition

Context Switching is the process of saving the state of current process and loading the state of another process.

It happens when time quantum expires, interrupt occurs, or process goes to waiting state.

Context Switch Simulation

CPU Currently Running:
P1
CPU Registers

SP = 500
ACC = 42
Step 1: CPU is executing Process P1.

Context Switching Steps

  1. Interrupt occurs (timer / I/O / system call)
  2. OS saves registers + program counter in PCB of old process
  3. OS changes state: Running â†’ Ready/Waiting
  4. Scheduler selects next process
  5. OS loads registers + PC from PCB of new process
  6. CPU continues execution from saved point

Why Context Switching is Expensive?

  • CPU time wasted in saving/loading registers
  • Cache memory becomes less efficient
  • More switching = more overhead

Inter-Process Communication (IPC)

What is IPC?

IPC is the mechanism that allows processes to communicate and share data.

Without IPC, processes will work separately and cannot cooperate.

1) Pipes

Used for communication between parent-child process.

Example: Linux command output â†’ input

ls | grep

2) Message Queue

Processes send messages like a mailbox.

Example: Chat application server messages.

3) Shared Memory

Processes share same memory region.

Fastest IPC method.

4) Sockets

Used for network communication.

Example: Browser talking to web server.

5) Signals

Used to notify process.

Example: CTRL + C kills process.

6) Semaphores

Used for synchronization.

Controls access to shared resource.

Real-life Analogy

  • Pipes: Water pipe between two houses
  • Message Queue: Post office letters
  • Shared Memory: Shared notebook in classroom
  • Socket: Phone call between two cities

ðŸ’¡ Fun Fact: WhatsApp uses sockets + message queue internally.

Threads (Concept)

What is a Thread?

A Thread is the smallest unit of CPU execution inside a process.

  • Process = Big container
  • Thread = Worker inside process

Process vs Thread

Feature Process Thread
Memory Separate memory Shared memory
Creation Cost High Low
Switching Slow (context switch) Fast
Communication IPC needed Easy (shared memory)

Real Example

  • Chrome process has multiple threads
  • One thread for UI
  • One thread for downloading
  • One thread for video playback

Types of Threads

1) User-Level Threads (ULT)

  • Managed by user thread library
  • Kernel does not know about threads
  • Faster creation and switching
  • But if one thread blocks, entire process blocks

2) Kernel-Level Threads (KLT)

  • Managed by operating system kernel
  • Kernel schedules threads directly
  • More overhead
  • But better concurrency

Exam Point (Very Important)

ULT is fast but has blocking problem.
KLT is powerful but overhead is high.

Thread Libraries

What is Thread Library?

A Thread Library provides functions to create and manage threads.

Programmer uses library functions, OS executes them.

Common Thread Libraries

  • Pthreads (POSIX): Used in Linux / UNIX systems
  • Windows Threads: Used in Microsoft Windows OS
  • Java Threads: Used in Java language (JVM handles)

Real Example

  • Linux server uses pthreads for web server requests
  • Java uses threads for Android apps
  • Games use multi-threading for graphics + physics

Thread Issues (Very Important)

1) Race Condition

Two threads accessing shared data at same time causes wrong output.

Example: Two people updating same bank account balance.

2) Deadlock

Two threads waiting for each other forever.

Example: Train A waiting for Track B, Train B waiting for Track A.

3) Starvation

Thread waits for CPU for long time due to priority scheduling.

4) Synchronization

Synchronization is controlling access to shared resource.

  • Semaphore
  • Mutex Lock
  • Monitor

ðŸ’¡ Interesting Fact: Many real software bugs happen because of race conditions.
